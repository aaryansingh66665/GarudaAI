Privacy Architecture & Technical Specs

Read how GarudaAI evaluates content locally without network leak hazards.

The "Zero Cloud" Privacy Pact

Traditional anti-phishing products inspect links or emails by sending them to web APIs. If the user is viewing confidential records, financial statements, or personal chats, this leaks sensitive information to service vendors.

"GarudaAI is built on the principle that threat inspection should never require information disclosure. All heuristics, OCR character mapping, and Transformer classification tasks are executed exclusively inside the browser environment."

How the Scanning Pipeline Operates Offline:

1. Input Hook

Characters staged strictly inside the DOM.

2. ONNX Inference

MiniLM model executes on client JS thread.

3. Local Report

Risk values calculated and loaded in view.

FAQ

Q: Is my data safe from browser extensions?

Yes. By running strictly inside standard sandboxed memory, GarudaAI doesn't transmit network traffic that background monitors could intercept. Data resides exclusively in temporary JS variables or your sandbox IndexedDB.

Q: How are new phishing patterns updated offline?

The Service Worker updates heuristics rules and local models whenever the application is opened online. Once the models are refreshed, they remain cached for fully offline verification checks.
